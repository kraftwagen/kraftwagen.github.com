Getting started with Kraftwagen

Using Kraftwagen in your own projects is fairly easy, especially if you are a seasoned Drupal developer already. This page is aimed at developers that are already familiar with Drupal.

Installation

To install Kraftwagen, you will need to install Drush and install Kraftwagen at a location that Drush can reach it. Drush can be installed in your home directory or globally. Take a look at the Drush documentation for more information about this. After you installed Drush you can install Kraftwagen as any other extension in the global directory (/usr/share/drush/commands) or in your home directory (~/.drush).

Per project

If you prefer to keep a Drush and Kraftwagen installation per project, Kraftwagen provides a little tool for that. Change to the directory your project in located in (not the src dir, but the project root directory). Take a look at our Kraftwagen Tools repository.

Create a new project

Creating a new project is done using the drush kw-new-project command. This command uses a so called skeleton to kickstart a project. A skeleton for Kraftwagen is a Git repository that contains the starting point for a project. Kraftwagen provides an example skeleton which you can fork to create your own.

Start by creating a directory for your project and changing to this directory. Then run drush kw-new-project in this directory. This command will ask for the human readable name of the project and the machine name of the project. Then it will ask for the location of the skeleton repository and which branch to us from the repository. When you supplied all the information, it will clone the skeleton repository and create a src directory in your current directory that contains the source of the project that is generated from the skeleton.

Now the source of the project is created. We advise you to use your favorite version control system to create the initial commit of your project.

You're now at the same state as after checking out an existing project.

Setting up existing projects

To get started on an existing project, you'll need to setup the project on your computer. First create a directory for the project and change to that directory. Then checkout the source code from the version control system it is saved in. Make sure the checkout is in the src directory.

Now run drush kw-setup. This will create a cnf directory with settings files, an environment file and a files directory. You should change the settings.local.php file to match your settings. You should choose the correct database credentials in this file. Furthermore, if you want to use the development settings, you should changes the contents to the enviroment file to the 'development' (omit the quotes).

You're now ready to create the first build of the project.

Creating a build

The source of the project is not working Drupal installation. It doesn't even contain Drupal itself. A working Drupal installation that is generated from the source is called a 'build'.

By default the build is created in a directory build. If you prefer to keep a directory of all builds and prefer build to be a symlink to the current build, you only have to create a directory builds. Kraftwagen will automatically create the build there if it exists and create a symlink for you.

A build is created by running drush kw-build.
